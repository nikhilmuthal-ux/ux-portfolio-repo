02 Core Interaction · Progressive Disclosure

Info  ⇄  less

The defining interaction. One toggle collapses the header to a calm identity line, or expands it to full context: runtime, metadata and controls.

SHOWN BELOW: EXPANDED STATE · the live build toggles between the two
COLLAPSED · “INFO”

Identity only: breadcrumb, status, title, description. The default. Calm, compact, and it gives the screen back to the work.

EXPANDED · “LESS”

Full context on demand: instance, version, space, profile, REST doc URL, job counts and the monitor toggles. One click away, never in the way.

07 Component Model

Composed from functions

The visible anatomy maps one-to-one to the code. The metadata panel only mounts when expanded.

<AppHeader state={collapsed | expanded}>
├─ <ContextBar> // RegionSelector · UserMenu
├─ <Breadcrumb /> // chevron hierarchy chips
├─ <TitleRow> // StatusDot · Title · Actions
├─ <DescriptionRow> // InfoToggle · EditableText
└─ <MetadataPanel visible={isExpanded}>
├─ <MetaField /> // instance · version · space · profile
├─ <JobStatusGroup /> // created · running · faulted…
└─ <ToggleGroup /> // monitor · instrumentation
